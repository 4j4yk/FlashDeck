# Support

## Where To Ask For Help

Default public support path:

- GitHub Issues
  - reproducible bugs
  - broken release artifacts
  - incorrect bundled content
  - import/export problems
  - deck or knowledge format questions
  - roadmap proposals

If GitHub Discussions is enabled later, longer-form design and community deck conversations can move there. Until then, use Issues.

## What To Include

For bug reports, include:

- device model or simulator
- iOS version
- app version or commit SHA if building locally
- import file used, if relevant
- exact reproduction steps
- screenshot or screen recording if the issue is visual

## Deck And Knowledge Contributions

When reporting content issues or proposing deck improvements, include:

- deck ID
- card ID or knowledge document ID
- why the content is inaccurate or unclear
- a proposed replacement if possible

## Security Issues

For vulnerabilities, follow `SECURITY.md` instead of posting exploit details publicly.
